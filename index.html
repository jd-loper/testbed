Types of Data

  • number: numbers of any kind.
  • bigint: big numbers of arbitrary length.
  • string: for strings with zero or more characters.
  • boolean: for true/false statements.
  • null: for unknown or empty values.
  • undefined: for unassigned values.
  • object: for complex data structures.
  • symbol: for unique identifiers.

Comparisons

  • Comparison operators return a boolean value.
  • Strings are compared letter-by-letter.
  • When values of different types are compared, they are converted to numbers.
  • The values null and undefined equal == each other and do not equal any other value.
  • A strict equality operator === checks the equality without type conversion.
  • Comparison operators:
    • == equal to.
    • === equal value and type.
    • != not equal.
    • !== not equal value and type.
    • > greater than.
    • < less than.
    • >= greater than or equal to.
    • <= less than or equal to.

Conditionals

  • Use if to specify a block of code to be executed, if a specified condition is true.
  • Use else to specify a block of code to be executed, if the same condition is false.
  • Use else if to specify a new condition to test, if the first condition is false.
  • Use switch to specify many alternative blocks of code to be executed.
  • The ternary operator ? may be used in place of longer if and else statements.
  • The ternary operator ? takes three arguments:
    • A comparison argument.
    • A result upon a true comparison.
    • A result upon a false comparison.

Logical Operators

  • The logical operator OR || can be used in an if statement to test if any given condition is true.
  • OR || will evaluate operands from left to right and return the first truthy value, if given multiple || values.
  • If all operands are false, only the last operand will be returned.
  • AND && returns true if both operands are truthy, and false otherwise.
  • Given multiple && values, only the first falsy value will be returned.
  • AND && has higher precedence than OR ||.
  • NOT ! converts an operand to boolean type true or false and returns the inverse value.
  • NOT ! has the highest precedence of all logical operators.

Functions

  • Functions are reusable blocks of code.
  • A function can be called whenever needed instead of rewriting the code.
  • Invoking a function means to run, or execute.
  • Some functions require parameters (values within the function parentheses).
  • Variables and other code defined inside a function are within their own scope.
  • Variables defined outside of all functions are in the global scope.
  • Arguments are values that are passed to the function.
